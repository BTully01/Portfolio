Zool Redimensioned

Project Status: Shipped Game

Development Team Size: 11

Originally released in 1992 on the Amiga, Zool is a 2D platformer that captivated audiences with its precise controls, vibrant design, and challenging difficulty.

Zool Redimensioned is a modern remaster of the classic game, retaining the original graphics while enhancing gameplay with updated mechanics and smoother controls.

The game was developed entirely in C++ using a proprietary in-house game engine.

My Contributions

I worked on the secret game levels, accessible through the main game, where I recreated the player class and enemy AI. This involved faithfully replicating classic enemy behaviors and refining player controls to maintain the original feel of the game.

Additionally, I contributed to the PlayStation release of the game, where I primarily designed the architecture for the party game modes. I created a flexible base class and a game mode manager to ensure consistency across different party modes. This system managed shared variables such as player scores, controller assignments, and other game-wide data.

I also led the implementation of PlayStation trophies, adapting the existing Steam achievement framework to support PlayStation's requirements. This involved developing a centralized trophy management system that allowed various parts of the game to trigger trophy unlocks. I collaborated closely with artists to create trophy artwork that met PlayStation's technical specifications.

Post-release, I worked with the University of Sheffield to integrate analytics into Zool Redimensioned. I designed a centralized framework to collect gameplay data, package it into an HTTP request, and send it to a Sumo-owned server for analysis. Given the sensitive nature of data collection, I ensured compliance with GDPR by implementing a system that generated unique but non-identifiable user IDs, allowing data tracking without compromising user privacy.

A research paper was published about the findings of the data collected through these analytics. Here is a link to the research paper.

Oil Strike 75

Project Status: Shipped Game

Development Team Size: 11

Oil Strike 75 was developed as a promotional game to tease the release of the then-upcoming title, Still Wakes the Deep. The game was built entirely in C++ using a proprietary in-house game engine and intergrated with the SteamAPI for its Steam functionality and offical release on the Steam marketplace.

My Contributions

I was responsible for implementing Steam achievements into the game. This involved collaborating with the designers of Still Wakes the Deep to ensure the achievements aligned with those in the main game. Additionally, the achievements were designed to serve as player statistics, tracking engagement metrics such as level drop-off rates.

My role required working closely with other programmers to understand different aspects of the game and determine the best locations in the codebase to track achievement progress. I also integrated the Steam API into the project, considering its implementation across various parts of the game.

Beyond programming, I collaborated with artists to create achievement icons that met Steam's technical requirements. I was responsible for uploading these assets to SteamWorks, managing achievement configurations, and ensuring seamless integration. Additionally, I created and managed private testing branches of the game, allowing QA testers to validate achievement functionality before release.

Plant It

Project Status: Game Jam

Development Team Size: 5

Plant It is a small project I worked on as part of a team of programmers. Inspired by crazy golf, the game aimed to replicate a similar feeling while also allowing us to experiment with advanced algorithms in a real-time environment.

My Contributions

My primary focus was developing the collision system for the game, particularly the interaction between the ball (represented as a seed) and the terrain. Given that the ground featured varied and complex terrain, I implemented a Ray-Triangle Intersection (RTI) algorithm to handle precise collisions between the ball and the environment.

To optimize performance and ensure efficient collision detection, I utilized Axis-Aligned Bounding Boxes (AABB) for interactions between the ball and other game objects. This approach significantly reduced computational overhead, as testing every individual triangle in high-polygon models with RTI would have been impractical. By combining RTI for ground collisions and AABB for object interactions, I created a streamlined and performant collision system that maintained accuracy while keeping processing costs manageable.

This project allowed me to refine my understanding of collision detection techniques and optimization strategies in game development while contributing to a creative and engaging gameplay experience.

Spyder Design Patterns

Project Status: Learning Material

Development Team Size: 1

I was tasked with adapting a small game to demonstrate object-oriented design principles for a group of new apprentices.

For this, I restructured the original Spyder game, originally a simple project from the PlayBuffer manual, by organizing the code into an object-oriented design pattern. I introduced classes for each game object, ensuring a clear and modular structure that highlighted best practices in software design.
